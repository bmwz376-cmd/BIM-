# BIM教科書シリーズ 図解仕様書

## 📋 概要

このドキュメントは、外部ツール（Canva/Figma/PowerPoint等）で高品質な図解を作成するための詳細仕様書です。

**目的**: BIM利用技術者試験の受験者向けに、プロフェッショナルで分かりやすい図解を提供する

**対象図表**: 全15種類

**推奨サイズ**: 1920×1080px (16:9) または 1600×1200px (4:3)

**出力フォーマット**: PNG (150 DPI以上)

---

## 🎨 デザインガイドライン

### カラーパレット

| 用途 | カラーコード | 色名 | 使用箇所 |
|------|-------------|------|----------|
| **プライマリ** | `#2E86AB` | 深い青 | メイン要素、タイトル |
| **セカンダリ** | `#A23B72` | 紫 | 補足要素、強調 |
| **成功** | `#06A77D` | 緑 | OK例、正解 |
| **警告** | `#F18F01` | オレンジ | 注意事項 |
| **危険** | `#C73E1D` | 赤 | NG例、エラー |
| **情報** | `#4CC9F0` | 水色 | 補足情報 |

### 背景色（淡色）

| 用途 | カラーコード | 使用箇所 |
|------|-------------|----------|
| 淡い青 | `#E3F2FD` | 形状情報、基本要素 |
| 淡い緑 | `#E8F5E9` | 属性情報、成功例 |
| 淡い黄 | `#FFF9C4` | 関係情報、注意事項 |
| 淡い赤 | `#FFEBEE` | エラー例、問題点 |
| 淡い紫 | `#F3E5F5` | 補足情報、派生物 |

### フォント設定

- **タイトル**: 28-32pt, 太字 (Bold)
- **見出し**: 20-24pt, 太字 (Bold)
- **本文**: 16-18pt, 通常
- **説明文**: 14-16pt, 通常
- **注釈**: 12-14pt, イタリック

**推奨フォント**:
- 日本語: Noto Sans JP, ヒラギノ角ゴ, メイリオ
- 英数字: Roboto, Open Sans, Arial

### レイアウト原則

1. **余白**: 各要素間に十分な余白（最低20-30px）
2. **整列**: グリッドシステムを使用（8px or 16px基準）
3. **階層**: 視覚的階層を明確に（タイトル > 見出し > 本文）
4. **コントラスト**: 背景と文字の十分なコントラスト（WCAG AA基準）
5. **一貫性**: 全図表で統一されたスタイル

### 1. CAD vs BIM比較図
**ファイル名**: `cad_vs_bim.png`  
**サイズ**: 1920×1080px

**レイアウト**:
```
┌─────────────────┬─────────────────┐
│  従来のCAD      │  BIM            │
│  (左半分)       │  (右半分)       │
├─────────────────┼─────────────────┤
│ 平面図 (個別)   │                 │
│ 立面図 (個別)   │   [BIMモデル]   │
│ 断面図 (個別)   │   (中央の円)    │
│                 │   ↓  ↓  ↓      │
│ ⚠️ 個別管理    │  自動生成図面    │
│   整合性困難    │   ✓ 整合性保証  │
└─────────────────┴─────────────────┘
```

**要素**:
- 左側（CAD）: 赤系の配色、個別ファイルのアイコン、バツ印
- 右側（BIM）: 青・緑系の配色、中央に3Dモデルのイラスト、チェックマーク
- 矢印: 太めの矢印で「自動生成」を強調

**キーメッセージ**:
- CAD: 「図面ごとに個別管理 → 整合性の確保が困難」
- BIM: 「単一の情報源 → 自動生成・整合性保証」

### 2. BIM情報の3層構造
**ファイル名**: `info_layers.png`  
**サイズ**: 1600×1200px

**レイアウト**:
```
┌─────────────────────────────────────┐
│    BIMモデルの3層構造               │
├─────────────────────────────────────┤
│  ┌──────────────────────┐           │
│  │ 形状情報 (Geometry)  │  詳細説明 │
│  │ 淡い青背景           │  →       │
│  └──────────────────────┘           │
│           ↓                         │
│  ┌──────────────────────┐           │
│  │ 属性情報 (Property)  │  詳細説明 │
│  │ 淡い緑背景           │  →       │
│  └──────────────────────┘           │
│           ↓                         │
│  ┌──────────────────────┐           │
│  │ 関係情報 (Relationship)│ 詳細説明│
│  │ 淡い黄背景           │  →       │
│  └──────────────────────┘           │
└─────────────────────────────────────┘
```

**要素**:
- 3つの大きなボックス（角丸）
- 各ボックス内にアイコン（立方体、タグ、リンク等）
- 右側に箇条書きで詳細説明
- 下部に「統合してBIMモデルを構成」のメッセージ

**詳細説明の内容**:
- 形状情報: 3次元形状データ、寸法（長さ・幅・高さ）、位置・配置情報
- 属性情報: 材質・仕上げ、コスト・性能値、メーカー・型番
- 関係情報: 部材間の接続、階層構造、依存関係

## 🎯 制作時の注意点

### 必須要件
1. **高解像度**: 最低150 DPI（印刷品質）
2. **読みやすさ**: フォントサイズは最低14pt以上
3. **配色一貫性**: 指定のカラーパレットを使用
4. **余白**: 要素間に十分な余白を確保
5. **日本語対応**: 日本語フォントを正しく設定

### 推奨事項
1. **アイコン使用**: Font Awesome、Material Icons等を活用
2. **グラデーション**: 淡い色のグラデーションで奥行き表現
3. **影効果**: 適度なドロップシャドウで立体感
4. **矢印**: 太めの矢印（最低3px）でフローを明確化
5. **整列**: グリッドシステムで要素を正確に配置

### 避けるべき事項
1. **小さい文字**: 12pt未満のテキスト
2. **低コントラスト**: 背景と文字の色が近い
3. **過度な装飾**: 情報を邪魔する装飾
4. **不統一**: 各図で異なるスタイル
5. **低解像度**: 72 DPI以下の画像
