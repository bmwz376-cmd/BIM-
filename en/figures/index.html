    Diagram List¶

    1. CAD vs BIM Comparison¶

    File: cad_vs_bim.png (1.1MB)
    Purpose: Visualizing the fundamental differences between traditional CAD and BIM

    Shows: - Traditional CAD: Separate drawings (floor plans, elevations, sections) - BIM: Single information source generating all outputs automatically - Key difference: Manual vs automated, inconsistency risk vs guaranteed consistency

    2. 3-Layer BIM Information Structure¶

    File: info_layers.png (827KB)
    Purpose: Understanding the three-layer architecture of BIM information

    Layers: 1. Geometry: 3D shapes, dimensions, placement 2. Properties: Materials, specifications, performance, cost 3. Relationships: Connections between components, hierarchy, dependencies

    3. Building Lifecycle Flow¶

    File: lifecycle_flow.png (753KB)
    Purpose: BIM utilization throughout the entire building lifecycle

    Phases: 1. Planning: Feasibility studies, site analysis 2. Design: Architectural, structural, MEP design 3. Construction: Construction planning, scheduling, clash detection 4. Operation: Facility management, maintenance, renovation 5. Demolition: Record keeping, material recycling

    4. LOD Matrix¶

    File: lod_matrix.png (944KB)
    Purpose: Understanding Level of Development stages

    Matrix showing: - LOD 100: Conceptual (symbols, basic info, planning use) - LOD 200: Approximate (rough geometry, general specs, schematic design) - LOD 300: Precise (accurate geometry, detailed specs, design development) - LOD 400: Fabrication (fabrication-ready, shop drawings, construction) - LOD 500: As-Built (field-verified, actual information, operations)

    5. Element Structure Diagram¶

    File: element_structure.png (750KB)
    Purpose: Hierarchical structure of BIM elements

    Hierarchy: - Project - Site / Building / Other - Site / Floors / Rooms / MEP - Walls / Columns / Floors / Ceilings

    6. OpenBIM & IFC¶

    File: openbim_ifc.png (909KB)
    Purpose: Understanding software interoperability through IFC

    Shows: - Top: BIM software (Revit, ArchiCAD, Tekla) - Middle: IFC (Industry Foundation Classes) - neutral format - Bottom: Analysis/design software (structural, MEP, construction management) - Concept: Different software sharing data seamlessly

    7. 4D/5D BIM Concept¶

    File: 4d_5d_bim.png (925KB)
    Purpose: Multi-dimensional BIM beyond 3D

    Dimensions: - 3D: Base BIM model (geometry + properties) - 4D: + Time (construction simulation, scheduling) - 5D: + Cost (cost management, budgeting, estimation) - 6D/7D: + Facilities management, sustainability

    8. BEP Flow¶

    File: bep_flow.png (773KB)
    Purpose: BIM Execution Plan development process

    8 Steps: 1. Project goals definition 2. BIM use purpose clarification 3. Team structure & role assignment 4. LOD & deliverables definition 5. Data exchange & sharing methods 6. Software & environment setup 7. Quality control & verification methods 8. Operation start & review

    9. Worksharing Concept¶

    File: worksharing_concept.png (810KB)
    Purpose: Understanding collaborative BIM workflows

    Shows: - Central Model (server): Central file repository - Multiple Users: Designer A (Architectural), Designer B (Structural), Designer C (MEP) - Synchronization: Continuous sync and integration - Benefit: Multiple users working simultaneously

    10. Family Hierarchy¶

    File: family_hierarchy_detail.png (770KB)
    Purpose: Revit family system structure

    Three Types: 1. System Families: Walls, floors, ceilings (built-in) 2. Loadable Families: Windows, doors, furniture (user-created) 3. In-Place Families: Special custom geometry

    11. Clash Detection¶

    File: clash_detection.png (820KB)
    Purpose: Interference checking in BIM

    Shows: - Conflict: Structural beam intersecting with MEP pipe - Detection: Automatic identification with warning - Result: Issue reported with suggested solutions - Benefit: Problems discovered and resolved during design phase

    12. Level Setting Mistakes (NG/OK)¶

    File: ng_ok_level_mistake.png (925KB)
    Purpose: Correct vs incorrect level naming

    Comparison: - ‚ùå NG: FL+3000, FL+6000, FL+9000 (confusing, error-prone) - ‚úÖ OK: 1FL (FL¬±0), 2FL (FL+3000), 3FL (FL+6000), RF (FL+9000) (clear hierarchy)

    13. Level Setting Details¶

    File: level_mistake_detail.png (858KB)
    Purpose: Detailed procedures for level settings

    5 Steps: 1. Level naming rules (1FL, 2FL, 3FL, RF) 2. Base level setup (1FL = FL¬±0) 3. Floor height setting (Standard: 3000mm) 4. Adding levels (bottom to top: 1FL ‚Üí 2FL ‚Üí 3FL) 5. Verification (no duplicates, elevation consistency, view range)

    14. Wall Placement Mistakes¶

    File: wall_mistake_patterns.png (970KB)
    Purpose: Common wall placement errors

    3 Mistake Patterns: - Mistake 1: Wrong level (walls on different floors) - Mistake 2: Reference point offset (not aligned to grid) - Mistake 3: Wall junction gaps (incomplete connections)

    Correct Method: - Verify level before placement - Align to grid lines - Ensure proper wall junctions
